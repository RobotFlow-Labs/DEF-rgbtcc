# PRD-002 — Dataset + Checkpoint Contract Validation

## Goal
Validate required data and checkpoint assets before training/eval.

## Inputs
- `scripts/server/validate_assets.py`
- dataset root path
- checkpoint path

## Steps
1. Run asset validator with sample checks.
2. Confirm split directories: train/val/test.
3. Confirm sampled id pairs have `_RGB.jpg`, `_T.jpg`, `_GT.npy`.
4. Confirm checkpoint file exists.
5. Capture failures and path fixes.

## Outputs
- pass/fail validation report.

## Acceptance Criteria
1. Validator exits `0`.
2. No missing split or sampled pair mismatch.
3. Checkpoint path validated.
