# PRD-006 — CUDA Profiling and Hotspot Lock

## Goal
Identify bottlenecks and lock implementation order for kernel work.

## Inputs
- baseline benchmark results
- profiler traces (torch profiler/nsys)

## Steps
1. Profile end-to-end inference on representative input sizes.
2. Quantify per-op time share.
3. Rank optimization targets by wall time and complexity.
4. Confirm kernel order and expected speedups.

## Outputs
- hotspot report.
- approved optimization backlog.

## Acceptance Criteria
1. Top-3 bottlenecks identified with evidence.
2. Kernel roadmap approved for PRD-007.
