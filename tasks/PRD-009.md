# PRD-009 — Dual-Compute Validation Report

## Goal
Validate and document CUDA vs MLX behavior and performance.

## Inputs
- CUDA optimized path
- MLX parity outputs

## Steps
1. Run matched-input inference on both backends.
2. Compute numeric differences with tolerance bands.
3. Compare latency/throughput.
4. Produce final validation report.

## Outputs
- dual-compute report artifact.

## Acceptance Criteria
1. Numeric drift is measured and explained.
2. Performance deltas are quantified.
